関数の構文

パターンマッチ

この章では Haskell のクールな構文をいくつか取り上げます。最初はパターンマッチです。パターンマッチは、データが従うべきパターンを指定し、データがそれに合致するか調べ、合致するならパターンに従ってデータを分解する仕組みです。

関数を定義するとき、パターンごとに別々の関数本体を定義できます。これによってシンプルで読みやすいコードが書けます。パターンマッチは任意のデータ型に使えます — 数値、文字、リスト、タプル、などなど。手始めに、与えられた数値が 7 かどうかを調べるごく簡単な関数を作ってみましょう。

lucky :: (Integral a) => a -> String
lucky 7 = "LUCKY NUMBER SEVEN!"
lucky x = "Sorry, you're out of luck, pal!"

lucky を呼ぶと、パターンが上から順にチェックされ、合致したパターンに対応する関数本体が使われます。最初のパターンに合致するのは 7 のときだけ。そうでなければ次のパターンに落ちて、何にでも合致して x に束縛されます。この関数は if 文でも実装できますが、たとえば 1 から 5 までの数を読み上げ、それ以外の数には "Not between 1 and 5" を返す関数を作りたいときは？　パターンマッチがなければ、入り組んだ if-then-else のツリーになります。パターンマッチを使えば：

sayMe :: (Integral a) => a -> String
sayMe 1 = "One!"
sayMe 2 = "Two!"
sayMe 3 = "Three!"
sayMe 4 = "Four!"
sayMe 5 = "Five!"
sayMe x = "Not between 1 and 5"

もし最後のパターン（catch-all = 何でも捕まえるパターン）を一番上に持ってきていたら、常に "Not between 1 and 5" を返してしまいます。すべての数を最初に捕まえてしまい、後ろのパターンに到達するチャンスがないからです。

前章で実装した階乗関数を覚えていますか？　数 n の階乗を product [1..n] と定義しました。階乗は 再帰的に も定義できます。これは数学でよく使われる定義です。まず 0 の階乗は 1 と決めます。次に、任意の正の整数の階乗を「その整数 × 一つ前の数の階乗」と決めます。これを Haskell で書くとこうなります。

factorial :: (Integral a) => a -> a
factorial 0 = 1
factorial n = n * factorial (n - 1)

これが私たちが再帰的に関数を定義した最初の例です。再帰は Haskell ではとても重要で、後で詳しく見ます。要するに、たとえば 3 の階乗を求めようとすると、まず 3 * factorial 2 を計算しようとします。2 の階乗は 2 * factorial 1 なので、ここまでで 3 * (2 * factorial 1)。factorial 1 は 1 * factorial 0 なので 3 * (2 * (1 * factorial 0)) となります。さて種明かしです — 0 の階乗は 1 と定義しており、catch-all のパターンより先にこのパターンに合致するので、ただ 1 を返します。最終的な結果は 3 * (2 * (1 * 1)) と等価になります。もし2つ目のパターンを最初のパターンより上に置いていたら、0 を含むすべての数を捕まえてしまい、計算は永遠に終わらなかったでしょう。だからパターンの順序は重要で、より具体的なパターンを先に、汎用的なものを後に書くのが基本です。

パターンマッチは失敗することもあります。次のような関数を定義したとして： 

charName :: Char -> String
charName 'a' = "Albert"
charName 'b' = "Broseph"
charName 'c' = "Cecil"

想定外の入力で呼び出すと、こうなります：

ghci> charName 'a'
"Albert"
ghci> charName 'b'
"Broseph"
ghci> charName 'h'
"*** Exception: tut.hs:(53,0)-(55,21): Non-exhaustive patterns in function charName

網羅的でないパターンだと文句を言われます。当然です。パターンを書くときは、想定外の入力でプログラムがクラッシュしないように、必ず catch-all のパターンを含めるべきです。

パターンマッチはタプルにも使えます。2D 空間の2つのベクトル（ペア形式）を取って足し合わせる関数を作りたいときは？　ベクトルを足すには、x 成分どうし、y 成分どうしを足します。パターンマッチを知らなければ、こう書いていたでしょう：

addVectors :: (Num a) => (a, a) -> (a, a) -> (a, a)
addVectors a b = (fst a + fst b, snd a + snd b)

これでも動きますが、もっと良い書き方があります。パターンマッチを使うように書き換えましょう。

addVectors :: (Num a) => (a, a) -> (a, a) -> (a, a)
addVectors (x1, y1) (x2, y2) = (x1 + x2, y1 + y2)

はい、できあがり！　ずっと良くなりました。これは既に catch-all パターンになっています。addVectors の型は（どちらの場合も）addVectors :: (Num a) => (a, a) -> (a, a) - > (a, a) なので、引数として必ず2つのペアが渡されることが保証されています。

fst と snd はペアの要素を取り出します。トリプル用の関数はあるでしょうか？　標準では用意されていませんが、自分で作れます。

first :: (a, b, c) -> a
first (x, _, _) = x

second :: (a, b, c) -> b
second (_, y, _) = y

third :: (a, b, c) -> c
third (_, _, z) = z

_ はリスト内包表記のときと同じ意味で、「その部分は本当に気にしない」という意味です。だから単に _ と書きます。

そういえば、リスト内包表記でもパターンマッチが使えます。これを見てください：

ghci> let xs = [(1,3), (4,3), (2,4), (5,3), (5,6), (3,1)]
ghci> [a+b | (a,b) <- xs]
[4,7,6,8,11,4]

パターンマッチが失敗した場合は、単に次の要素に進むだけです。

リスト自身もパターンマッチに使えます。空リスト [] や、: と空リストを含む任意のパターンに合致させられます。[1,2,3] は 1:2:3:[] の糖衣構文なので、前者のパターンも使えます。x:xs のようなパターンは、リストの先頭を x に、残りを xs に束縛します。要素が1つしかない場合は xs は空リストになります。

Note: x:xs パターンはとてもよく使われます。特に再帰関数で。ただし : を含むパターンは長さ1以上のリストにしか合致しません。

たとえば最初の3要素を変数に束縛し、残りを別の変数に束縛したければ、x:y:z:zs のように書けます。これは要素3個以上のリストにのみ合致します。

リストに対するパターンマッチを覚えたところで、head 関数を自前で実装してみましょう。

head' :: [a] -> a
head' [] = error "Can't call head on an empty list, dummy!"
head' (x:_) = x

動くか確認：

ghci> head' [4,5,6]
4
ghci> head' "Hello"
'H'

いいね！　複数の変数に束縛したい場合（その中に実際は何にも束縛しない _ が含まれていても）、必ず括弧で囲む必要があることに注意してください。それから、ここで使った error 関数にも注目。文字列を取って実行時エラーを発生させ、その文字列をエラーの内容として表示します。プログラムをクラッシュさせるので、多用は禁物です。でも空リストに head を呼ぶのはそもそも意味をなしません。

リストの最初の数要素を、（不）便利な英語で告げる些細な関数を作ってみます。

tell :: (Show a) => [a] -> String
tell [] = "The list is empty"
tell (x:[]) = "The list has one element: " ++ show x
tell (x:y:[]) = "The list has two elements: " ++ show x ++ " and " ++ show y
tell (x:y:_) = "This list is long. The first two elements are: " ++ show x ++ " and " ++ show y

この関数は安全です。空リスト、要素1つのリスト、要素2つのリスト、要素3つ以上のリストすべてに対応しているからです。(x:[]) と (x:y:[]) はそれぞれ [x]、[x,y] と書き換えられます（糖衣構文なので括弧も不要）。(x:y:_) は長さ2以上の任意のリストに合致するので、角括弧では書き換えられません。

リスト内包表記を使って自前の length 関数は既に作りました。今度はパターンマッチとちょっとした再帰で書いてみましょう：

length' :: (Num b) => [a] -> b
length' [] = 0
length' (_:xs) = 1 + length' xs

これは前に書いた階乗関数と似ています。まず既知の入力（空リスト）の結果を定義します。これをエッジ条件 (edge condition) ともいいます。次に2つ目のパターンでリストを先頭と残りに分解します。長さは「1 + 残りの長さ」と書きます。先頭は気にしないので _ でマッチさせます。リストのすべてのパターンを網羅していることに注意。1つ目は空リスト、2つ目は空リスト以外のすべてのリストに合致します。

length' を "ham" で呼ぶと何が起こるか見てみましょう。まず空リストかどうか調べます。違うので2つ目のパターンに進みます。2つ目のパターンに合致して、長さは 1 + length' "am"（先頭と残りに分けて先頭を捨てる）と決まります。OK。"am" の length' は同じく 1 + length' "m"。今 1 + (1 + length' "m") となっています。length' "m" は 1 + length' ""（1 + length' [] でも同じ）。そして length' [] は 0 と定義しています。最終的に 1 + (1 + (1 + 0)) です。

sum も実装してみましょう。空リストの合計は 0。これをパターンとして書きます。リストの合計は「先頭 + 残りの合計」だと分かっています。書き起こすと：

sum' :: (Num a) => [a] -> a
sum' [] = 0
sum' (x:xs) = x + sum' xs

as パターン というものもあります。これはあるものをパターンに従って分解しつつ、全体への参照も保ちたいときに便利です。パターンの前に名前と @ を付けて書きます。たとえば xs@(x:y:ys)。このパターンは x:y:ys と同じものに合致しますが、関数本体で x:y:ys と繰り返して書く代わりに xs を使えるので楽です。手早く例を：

capital :: String -> String
capital "" = "Empty string, whoops!"
capital all@(x:xs) = "The first letter of " ++ all ++ " is " ++ [x]

ghci> capital "Dracula"
"The first letter of Dracula is D"

普通、as パターンは「より大きなパターンにマッチさせて、関数本体で全体ももう一度使う必要があるけれど、書き直したくない」というときに使います。

もう一点 — パターンマッチで ++ は使えません。(xs ++ ys) をパターンマッチに使おうとしても、何が最初のリストで何が2つ目のリストになるべきでしょう？　意味をなしません。(xs ++ [x,y,z]) や (xs ++ [x]) なら意味が通りそうですが、リストの本質上、これも使えません。

ガードでガード！

パターンが値の形を確認しつつ分解する方法だとすれば、ガードは値の何らかの性質（あるいは複数の性質）が真か偽かをテストする方法です。これは if 文に似ていて、実際とても似ています。違いは、複数の条件があるときガードのほうが格段に読みやすく、しかもパターンと相性がとても良いことです。

構文を説明するより、いきなりガードを使った関数を作ってみましょう。あなたの BMI（ボディマス指数）に応じて違う罵り方をするシンプルな関数を作ります。BMI は体重を身長の2乗で割った値です。BMI が 18.5 未満なら「やせ」、18.5 から 25 までは「普通」、25 から 30 は「やや肥満」、30 を超えると「肥満」とされます。さて関数（今は計算は別途。BMI を受け取ってあなたを罵るだけ）：

bmiTell :: (RealFloat a) => a -> String
bmiTell bmi
    | bmi <= 18.5 = "You're underweight, you emo, you!"
    | bmi <= 25.0 = "You're supposedly normal. Pffft, I bet you're ugly!"
    | bmi <= 30.0 = "You're fat! Lose some weight, fatty!"
    | otherwise   = "You're a whale, congratulations!"

ガードは関数名と引数のあとに続くパイプ記号で示します。普通は少し右にインデントして揃えます。ガードは要するに真偽値を返す式です。True に評価されれば対応する関数本体が使われ、False ならチェックは次のガードに進む、という具合です。この関数を 24.3 で呼ぶと、まず 18.5 以下かどうかを調べます。違うので次のガードへ。次のガードでは 24.3 が 25.0 より小さいので、2 つ目の文字列が返ります。

これは命令型言語の大きな if-else ツリーを思い出させますが、こちらのほうがずっと優れていて読みやすいです。大きな if-else ツリーは普通敬遠されますが、問題によってはどうしてもそうした離散的な定義が必要なこともあります。ガードはその素敵な代替です。

多くの場合、最後のガードは otherwise です。otherwise はただ otherwise = True と定義されており、何でも捕まえます。これはパターンに似ていますが、パターンが入力がパターンを満たすかをチェックするのに対し、ガードは真偽条件をチェックします。関数のすべてのガードが False に評価された場合（しかも otherwise の catch-all を用意していなかった場合）、評価は次の パターン へ落ちていきます。これがパターンとガードが仲良くプレイする仕組みです。適切なガードもパターンも見つからなければ、エラーが投げられます。

もちろんガードは何個の引数を取る関数にも使えます。ユーザーに自前で BMI を計算してから呼び出してもらう代わりに、身長と体重を受け取って計算もこちらでやるように関数を書き換えましょう。

bmiTell :: (RealFloat a) => a -> a -> String
bmiTell weight height
    | weight / height ^ 2 <= 18.5 = "You're underweight, you emo, you!"
    | weight / height ^ 2 <= 25.0 = "You're supposedly normal. Pffft, I bet you're ugly!"
    | weight / height ^ 2 <= 30.0 = "You're fat! Lose some weight, fatty!"
    | otherwise                 = "You're a whale, congratulations!"

私が太っているか確認してみましょう……

ghci> bmiTell 85 1.90
"You're supposedly normal. Pffft, I bet you're ugly!"

やった！　太ってない！　でも Haskell に醜いと言われた。まあいいや！

関数名と引数のあと、最初のガードの前に = がないことに注意してください。初心者はそこに = を入れてしまって構文エラーをよく起こします。

もう一つシンプルな例：自前の max 関数を実装してみましょう。これは比較できる2つの値を取り、大きいほうを返す関数でしたね。

max' :: (Ord a) => a -> a -> a
max' a b
    | a > b     = a
    | otherwise = b

ガードは1行で書くこともできますが、ごく短い関数でも読みにくくなるのでおすすめしません。一応、max' を1行で書くとこうなります：

max' :: (Ord a) => a -> a -> a
max' a b | a > b = a | otherwise = b

うえー！　全然読みやすくない！　次に進みましょう：自前の compare をガードで実装してみます。

myCompare :: (Ord a) => a -> a -> Ordering
a `myCompare` b
    | a > b     = GT
    | a == b    = EQ
    | otherwise = LT

ghci> 3 `myCompare` 2
GT
Note: 関数をバッククォートで中置として呼び出せるだけでなく、バッククォートで定義することもできます。そのほうが読みやすい場合もあります。

where!?

前のセクションで、BMI 計算と罵倒を行う関数をこう定義しました：

bmiTell :: (RealFloat a) => a -> a -> String
bmiTell weight height
    | weight / height ^ 2 <= 18.5 = "You're underweight, you emo, you!"
    | weight / height ^ 2 <= 25.0 = "You're supposedly normal. Pffft, I bet you're ugly!"
    | weight / height ^ 2 <= 30.0 = "You're fat! Lose some weight, fatty!"
    | otherwise                   = "You're a whale, congratulations!"

同じ式を3回繰り返しているのに気づきましたか。同じ式を3回繰り返しているのに気づきましたか。プログラミング中に同じことを（3回）繰り返すのは、頭を蹴られるくらい望ましくないことです。同じ式を3回繰り返しているので、一度計算して名前に束縛し、その名前を使えたら理想的です。次のように書き換えられます：

bmiTell :: (RealFloat a) => a -> a -> String
bmiTell weight height
    | bmi <= 18.5 = "You're underweight, you emo, you!"
    | bmi <= 25.0 = "You're supposedly normal. Pffft, I bet you're ugly!"
    | bmi <= 30.0 = "You're fat! Lose some weight, fatty!"
    | otherwise   = "You're a whale, congratulations!"
    where bmi = weight / height ^ 2

ガードのあとに where キーワードを置き（普通はパイプと同じくらいインデントするのが見やすい）、いくつかの名前や関数を定義します。これらの名前はガード全体から見え、繰り返しを避けられます。BMI の計算式を変えることになっても、一箇所変えるだけで済みます。読みやすさも増しますし、bmi のような変数は一度しか計算されないのでプログラムが速くなることもあります。さらにオーバーキル気味にこう書くこともできます：

bmiTell :: (RealFloat a) => a -> a -> String
bmiTell weight height
    | bmi <= skinny = "You're underweight, you emo, you!"
    | bmi <= normal = "You're supposedly normal. Pffft, I bet you're ugly!"
    | bmi <= fat    = "You're fat! Lose some weight, fatty!"
    | otherwise     = "You're a whale, congratulations!"
    where bmi = weight / height ^ 2
          skinny = 18.5
          normal = 25.0
          fat = 30.0

関数の where 節で定義した名前はその関数からしか見えないので、他の関数の名前空間を汚染する心配はありません。すべての名前が同じ列に揃っていることに注意してください。揃え方が雑だと、Haskell は同じブロックに属する定義だと認識できなくなります。

where 束縛は、別々のパターンの関数本体間では共有されません。1 つの関数の複数のパターンで共通の名前にアクセスしたければ、グローバルに定義する必要があります。

where 束縛で パターンマッチ もできます！　先ほどの関数の where 節はこう書き換えられます：

    ...
    where bmi = weight / height ^ 2
          (skinny, normal, fat) = (18.5, 25.0, 30.0)

もう少し些細な例を作ります。名と姓を受け取って、その人のイニシャルを返す関数です。

initials :: String -> String -> String
initials firstname lastname = [f] ++ ". " ++ [l] ++ "."
    where (f:_) = firstname
          (l:_) = lastname

このパターンマッチは関数の引数で直接行うこともできました（むしろそのほうが短くて分かりやすい）が、where 束縛でも可能だ、という例として示しました。

where ブロックには定数だけでなく関数も定義できます。健康プログラミングのテーマを引き継ぎ、体重と身長のペアのリストを取って BMI のリストを返す関数を作ってみましょう。

calcBmis :: (RealFloat a) => [(a, a)] -> [a]
calcBmis xs = [bmi w h | (w, h) <- xs]
    where bmi weight height = weight / height ^ 2

これだけです！　この例で bmi を関数として導入する必要があったのは、関数の引数だけから1つの BMI を計算するのではなく、関数に渡されたリストの各ペアごとに違う BMI を計算する必要があるからです。

where 束縛はネストもできます。関数の where 節で補助関数を定義し、その補助関数自身にも where 節で別の補助関数を定義する、というのはよくあるイディオムです。

let it be

where 束縛とよく似たものに let 束縛があります。where 束縛は、関数の末尾で変数を束縛して、関数全体（すべてのガードを含む）からそれらが見える、という構文です。一方 let 束縛はどこでも変数を束縛でき、それ自体が式ですが、とてもローカルなのでガードをまたいで見えません。Haskell の値を名前に束縛するすべての構文と同様、let 束縛もパターンマッチに使えます。動かしてみましょう！　円柱の表面積を高さと半径から求める関数はこう書けます： 

cylinder :: (RealFloat a) => a -> a -> a
cylinder r h =
    let sideArea = 2 * pi * r * h
        topArea = pi * r ^2
    in  sideArea + 2 * topArea

形は let <束縛> in <式>。let 部分で定義した名前は、in 部分の式から見えます。同じことは where 束縛でも書けます。名前が一列に揃っているのも同じです。両者の違いは何でしょう？　今のところは、let は束縛が先で、それを使う式が後、where は逆、という違いに見えます。

違いは、let 束縛は式そのものだということです。where 束縛は単なる構文要素です。前に if 文を扱ったとき、if-else は式なのでほぼどこにでも詰め込める、と説明したのを覚えていますか？

ghci> [if 5 > 3 then "Woo" else "Boo", if 'a' > 'b' then "Foo" else "Bar"]
["Woo", "Bar"]
ghci> 4 * (if 10 > 5 then 10 else 0) + 2
42

let 束縛も同じことができます。

ghci> 4 * (let a = 9 in a + 1) + 2
42

ローカルスコープに関数を導入するのにも使えます：

ghci> [let square x = x * x in (square 5, square 3, square 2)]
[(25,9,4)]

複数の変数を1行で束縛したいときは、当然列を揃えられません。なのでセミコロンで区切れます。

ghci> (let a = 100; b = 200; c = 300 in a*b*c, let foo="Hey "; bar = "there!" in foo ++ bar)
(6000000,"Hey there!")

最後の束縛のあとにセミコロンは必須ではありませんが、付けても構いません。前述のとおり、let 束縛でもパターンマッチが使えます。タプルをすばやく要素に分解して名前に束縛するのに便利です。

ghci> (let (a,b,c) = (1,2,3) in a+b+c) * 100
600

リスト内包表記の中にも let 束縛を入れられます。先ほどの「体重と身長のペアのリストから BMI のリストを計算する関数」を、where で補助関数を定義する代わりに let をリスト内包表記内で使うように書き換えましょう。

calcBmis :: (RealFloat a) => [(a, a)] -> [a]
calcBmis xs = [bmi | (w, h) <- xs, let bmi = w / h ^ 2]

let をリスト内包表記の中に置くのは、述語を置くのと似ていますが、let はリストをフィルタしません。名前を束縛するだけです。リスト内包表記内の let で定義された名前は、出力関数（| の前の部分）からも、let 束縛より後に来るすべての述語・セクションからも見えます。なので、太った人だけの BMI を返すように書き換えられます：

calcBmis :: (RealFloat a) => [(a, a)] -> [a]
calcBmis xs = [bmi | (w, h) <- xs, let bmi = w / h ^ 2, bmi >= 25.0]

bmi という名前は (w, h) <- xs の部分では使えません。なぜなら let 束縛より前に出てくるからです。

リスト内包表記で let 束縛を使うときは in 部分を省略しました。リスト内包表記では名前の可視範囲が既に決まっているからです。とはいえ、述語の中で let in 束縛を書くこともでき、その場合は名前はその述語の中だけで見えます。in 部分は、GHCi で関数や定数を直接定義するときも省略できます。その場合、定義した名前は対話セッション全体から見えるようになります。

ghci> let zoot x y z = x * y + z
ghci> zoot 3 9 2
29
ghci> let boot x y z = x * y + z in boot 3 4 2
14
ghci> boot
<interactive>:1:0: Not in scope: `boot'

let 束縛がそんなにクールなら、where 束縛の代わりにいつも let を使えばよいのでは？　と思うかもしれません。残念ながら、let 束縛は式であって、しかもスコープが結構ローカルなので、ガードをまたぐことができません。where を好む人もいます。where なら名前が関数の本体の後に来るので、関数本体が関数名と型宣言に近くなり、そのほうが読みやすいと感じる人もいるからです。

case式

多くの命令型言語（C、C++、Java、など）には case 構文があり、使ったことがあれば仕組みは想像がつくでしょう。変数を取って、その値ごとにコードブロックを実行する。設定していない値の場合に備えて catch-all のコードブロックを置くこともある、というやつです。

Haskell はその概念をさらに一段押し上げます。名前のとおり、case 式は文字どおり 式であり、if-else 式や let 束縛と同様です。変数の値によって式を評価できるだけでなく、パターンマッチもできます。ふむ、変数を取ってパターンマッチして値ごとに違うコードを評価する……どこかで聞きましたよね。そう、関数定義での引数のパターンマッチです！　あれは実は case 式の糖衣構文です。次の2つは同じ動きをし、互換可能です：

head' :: [a] -> a
head' [] = error "No head for empty lists!"
head' (x:_) = x

head' :: [a] -> a
head' xs = case xs of [] -> error "No head for empty lists!"
                      (x:_) -> x

見ての通り、case 式の構文はとてもシンプルです：

case expression of pattern -> result
                   pattern -> result
                   pattern -> result
                   ...

expression がパターンに対してマッチされます。マッチの動作は期待通り：式に最初に合致したパターンが使われます。case 式全体を抜けても合致するパターンがなければ実行時エラーが発生します。

関数引数でのパターンマッチは関数定義時にしかできませんが、case 式はほぼどこにでも使えます。たとえば：

describeList :: [a] -> String
describeList xs = "The list is " ++ case xs of [] -> "empty."
                                               [x] -> "a singleton list."
                                               xs -> "a longer list."

式の途中で何かにパターンマッチさせるのに便利です。関数定義でのパターンマッチは case 式の糖衣構文なので、これは次のように書くこともできます：

describeList :: [a] -> String
describeList xs = "The list is " ++ what xs
    where what [] = "empty."
          what [x] = "a singleton list."
          what xs = "a longer list."
